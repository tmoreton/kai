Getting Started

Installation

npm install
cp .env.example .env
# Add your OpenRouter API key to .env

npm run dev       # Development mode
npm run build     # Build for production
npm start         # Run built version

Quick Usage

# Start interactive REPL
kai

# One-shot query
kai "explain this codebase"

# Pipe input
echo "explain this" | kai
cat file.ts | kai "review this code"

# Start web UI + API
kai server

Memory System

Kai has three memory layers, all scoped per-project (auto-detected via .git, package.json, etc.).

Soul (Core Memory)

Persistent identity and per-project context. Always loaded into every conversation.

BlockScopePurpose
personaGlobalKai's identity and behavioral traits
humanGlobalWhat Kai knows about you (updated as it learns)
goalsPer-projectCurrent objectives for this project
scratchpadPer-projectWorking notes during multi-step tasks

Storage: ~/.kai/soul/ (global) and ~/.kai/projects/{id}/ (per-project)

Archival Memory

Long-term knowledge store for facts, preferences, research findings. Searchable by keyword and tags. Append-only JSONL format.

Storage: ~/.kai/projects/{id}/archival/knowledge.jsonl

Recall Memory

Searchable archive of past conversations across sessions. Keyword-based search returns matching messages with timestamps.

Storage: ~/.kai/projects/{id}/recall/history.jsonl

Background Agents

Agents run autonomous YAML workflows on a cron schedule. They can chain LLM calls, integrations, shell commands, and self-improvement review loops.

Workflow YAML Format

name: nightly-commit
description: Auto-commit config changes
schedule: "0 2 * * *"
config:
  custom_param: value

# Optional self-improvement loop
review:
  enabled: true
  max_iterations: 3

steps:
  - name: check_changes
    type: shell
    command: "cd ~/.kai && git status --porcelain"

  - name: generate_message
    type: llm
    prompt: "Generate a commit message for: ${vars.check_changes}"

  - name: save_data
    type: integration
    integration: data
    action: write
    params:
      file: "~/.kai/output.json"
      data: "${vars.generate_message}"

  - name: notify
    type: notify
    params:
      title: "Kai Agent"
      message: "Done!"

Step Types

TypeDescriptionKey Fields
llmSend a prompt to the LLMprompt
integrationCall a built-in integrationintegration, action, params
shellRun a shell commandcommand
notifySend a desktop notificationparams.title, params.message
reviewSelf-improvement review loopprompt

Configuration

Config is loaded from (highest priority first):

  1. .kai/settings.json (project-level)
  2. kai.config.json (project-level)
  3. ~/.kai/settings.json (user-level)

{
  "model": "moonshotai/kimi-k2.5",
  "mcp": {
    "servers": {
      "filesystem": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
        "env": {}
      }
    }
  },
  "permissions": {
    "mode": "default",
    "allow": [],
    "deny": []
  },
  "hooks": {
    "before": {},
    "after": {}
  }
}

Permissions

Tools have permission levels. In default mode, read-only tools are always allowed. Write operations and dangerous commands require confirmation.

  • default — Interactive confirmation for writes/deletes
  • auto — Approve everything (used in web/server mode)

Hooks

Run shell commands before or after any tool call. Configure in settings:

{
  "hooks": {
    "before": { "write_file": "echo 'About to write a file'" },
    "after": { "bash": "echo 'Command finished'" }
  }
}
