What can Kai do?

Your AI assistant that can chat, search the web, generate images, run automated tasks, and connect to your tools.

Chat Web Search Images Memory Agents Skills & MCP Commands Setup

Chat

Just type what you need. Kai understands natural language and figures out which tools to use — including reading and writing files on your computer.

Examples

  • "Write a blog post about our new product launch"
  • "Summarize this article" (paste a URL)
  • "Read marketing-plan.md and give me the key points"
  • "Generate a thumbnail for my YouTube video"
  • "Research trending topics in AI marketing"

Web Search

Kai can search the web and read web pages. Requires a Tavily API key for search. Fetching URLs works out of the box.

Examples

  • "Search for the latest trends in content marketing"
  • "Read this article and give me the key takeaways: https://..."
  • "Find recent news about [topic]"

Images

Generate images from text descriptions. Great for thumbnails, social posts, and concept art. You can also attach a reference photo for style matching.

Examples

  • "Generate a YouTube thumbnail showing a person excited about AI"
  • "Create a social media graphic with a futuristic city background"
  • "Make a hero image for a blog post about productivity"

Memory

Kai remembers things across conversations — your preferences, goals, and anything you tell it to save. Just say "remember this" or ask "what do you know about me?"

Examples

  • "Remember that our brand voice is casual and friendly"
  • "What did we decide about the launch date last time?"
  • "My goal for this week is to finish the email sequence"

Agents

Automated workflows that run on a schedule. Monitor competitors, research trends, generate ideas — all on autopilot. Switch to the Agents tab in the sidebar to manage them.

Common Use Cases

  • Daily trend research — Get a morning briefing on what's happening in your industry
  • Competitor monitoring — Track competitor launches and announcements
  • Content ideation — Generate fresh video/blog ideas weekly
  • Reporting — Pull stats and create weekly performance summaries

For Developers

Agents are defined as YAML workflow files with steps that can use AI, search YouTube, browse the web, read/write files, generate images, and run shell commands. Create custom agents for any repetitive workflow.

Skills & MCP — Connect Your Tools

Extend Kai by connecting external tools. Two ways to do this: Skills (simple JavaScript plugins) and MCP Servers (industry-standard tool connections).

🧩 What are Skills?

Skills add new capabilities to Kai through simple JavaScript files. Think of them like browser extensions — they give Kai new superpowers.

  • YouTube skill — Search videos, get stats, analyze channels
  • Web Tools — Fetch any URL, search with Tavily
  • Custom skills — Build your own for Slack, Notion, your API, etc.

To install: Go to Settings and paste a GitHub URL or local path. Skills live in ~/.kai/skills/.

🔌 What are MCP Servers?

MCP (Model Context Protocol) is an open standard for connecting AI to external tools. Thousands of MCP servers exist for databases, cloud services, APIs, and more. Kai can use any of them.

  • Filesystem access — Read/write files anywhere on your computer
  • Database queries — Connect to Postgres, MySQL, etc.
  • APIs — Connect to Stripe, Slack, GitHub, and hundreds more

To add one: Go to Settings → MCP Servers. You'll need the command to run the server (usually npx for Node-based servers).

For Developers: Creating a Skill

Skills are just a folder with 2 files. Create ~/.kai/skills/my-skill/ with:

# skill.yaml
id: my-skill
name: My Custom Skill
version: 1.0.0
tools:
  - name: hello
    description: Say hello
    parameters:
      name: { type: string, required: true }
// handler.js
export default {
  actions: {
    hello: async (params) => `Hello, ${params.name}!`,
  },
};

Custom Commands

Create custom commands by adding markdown files to .kai/commands/ (project-level) or ~/.kai/commands/ (global). The filename becomes the command name.

Example: Create .kai/commands/deploy.md with:

# Deploy to production

Run the deployment script with:
{{args}}

Always check tests pass first.

Then use /deploy --env=prod in chat.

npm run dev
